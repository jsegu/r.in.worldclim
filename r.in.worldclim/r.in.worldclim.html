DESCRIPTION

r.in.worldclim imports multiple files from the WorldClim current climate dataset. You first needs to download the layers first as zip files. Make sure to select the generic format (bil). After downloading, you can run r.in.worldclim to import the layers in your GRASS GIS database.

The script inflates requested files from their original archives before importing them as raster maps into GRASS. The data can be imported by tiles with tiles=... or globally with res=.... If tiles are imported, they are by default patched to create one layer per variable. The user can override this behaviour using the p flag.

NOTE

If you need data layers for a specific region, it is probably easiest to download the tiles covering that region. To find the tiles, you can use the r.in.worldclim.tiles addon, which finds the tiles that cover an user-defined region of interest and optionally downloads them.

EXAMPLES

Import global mean temperature at 10 arc-min resolution: 
r.in.worldclim.py fields=tmean res=10m

Import temperature and precipitation data for tile 16 (central Europe): 

r.in.worldclim.py fields=tmean,prec tiles=16
